# Security Policy

## Supported Versions

Security updates are applied to the following branches:

| Branch        | Supported          |
|---------------|--------------------|
| `main`        | Yes                |
| `development` | Yes (pre-release)  |
| Older branches| No                 |

## Reporting a Vulnerability

**Do not** open a public issue for a security vulnerability.

1. **Report privately**  
   Use one of the following:
   - **GitHub:** Go to the **Security** tab → **Advisories** → **Report a vulnerability** (if enabled for this repo).
   - **Email:** [jksahoo@goa.bits-pilani.ac.in](mailto:jksahoo@goa.bits-pilani.ac.in) or [mit4s.dev@gmail.com](mailto:mit4s.dev@gmail.com)

2. **Include:**
   - Description of the vulnerability and affected component (frontend, backend, config, etc.).
   - Steps to reproduce.
   - Impact (e.g. data exposure, privilege escalation).
   - Suggested fix or mitigation, if any.

3. **What to expect**
   - We will acknowledge receipt and aim to respond within a reasonable time.
   - We may ask for clarification.
   - We will work on a fix and coordinate disclosure (e.g. after a patch is released or after a short embargo).

Thank you for helping keep VisionAI and its users safe.
